# DTO in NestJS

DTO stands for Data Transfer Object.

A DTO is a class that defines the shape of data that should be sent between the client and the server.

A DTO is a class used to describe what data is allowed when creating, updating, or transferring data in an application.

Main purpose of DTO
- Defining request body structure
- Validating incoming data
- Making code more readable
- Controlling data flow

Where DTO is used in NestJS

Mostly DTO is used in:
- `@Body()`
- sometimes `@Query()`
- sometimes `@Param()`

example:

```
@Post()
createUser(@Body() createUserDto: CreateUserDto) {
  return this.userService.create(createUserDto);
}

```

Basic DTO example

example:

`create-user.dto.ts`

```
export class CreateUserDto {
  name: string;
  email: string;
  password: string;
}

```

`user.controller.ts`

```
import { Body, Controller, Post } from '@nestjs/common';
import { CreateUserDto } from './dto/create-user.dto';

@Controller('users')
export class UserController {
  @Post()
  create(@Body() createUserDto: CreateUserDto) {
    return createUserDto;
  }
}

```

It does not validate automatically unless we add:
  - class-validator
  - class-transformer
  - ValidationPipe


# DTO with validation

To make DTO powerful, NestJS uses decorators from class-validator.

Install packages if not installed:

```
npm install class-validator class-transformer

```

Example of validated DTO

`create-user.dto.ts`

```

import { IsEmail, IsNotEmpty, IsString, MinLength } from 'class-validator';

export class CreateUserDto {

  @IsString()
  @IsNotEmpty()
  name: string;

  @IsEmail()
  email: string;

  @IsString()
  @MinLength(6)
  password: string;

}

```

For DTO validation to work, you need to enable `ValidationPipe`.

`main.ts`

```

import { ValidationPipe } from '@nestjs/common';
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  app.useGlobalPipes(
    new ValidationPipe(),
  );

  await app.listen(3000);
}
bootstrap();

```

Full working example

example:

Step 1: DTO

```
import { IsEmail, IsNotEmpty, IsString, MinLength } from 'class-validator';

export class CreateUserDto {
  @IsString()
  @IsNotEmpty()
  name: string;

  @IsEmail()
  email: string;

  @IsString()
  @MinLength(6)
  password: string;
}

```

Step 2: Controller

```

import { Body, Controller, Post } from '@nestjs/common';
import { CreateUserDto } from './dto/create-user.dto';

@Controller('users')
export class UsersController {
  @Post()
  create(@Body() createUserDto: CreateUserDto) {
    return {
      message: 'User created successfully',
      data: createUserDto,
    };
  }
}

```

Step 3: main.ts

```
import { ValidationPipe } from '@nestjs/common';
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  app.useGlobalPipes(new ValidationPipe());

  await app.listen(3000);
}
bootstrap();

```

# DTO for query parameters

DTO is not only for body.
You can also use DTO for query.

Request:

`GET /users?page=1&limit=10`

```

import { IsInt, IsOptional, Min } from 'class-validator';
import { Type } from 'class-transformer';

export class UserQueryDto {
  @IsOptional()
  @Type(() => Number)
  @IsInt()
  @Min(1)
  page?: number;

  @IsOptional()
  @Type(() => Number)
  @IsInt()
  @Min(1)
  limit?: number;
}

```

Controller

```
import { Controller, Get, Query } from '@nestjs/common';
import { UserQueryDto } from './dto/user-query.dto';

@Controller('users')
export class UsersController {
  @Get()
  findAll(@Query() query: UserQueryDto) {
    return query;
  }
}

```

# DTO with optional and required fields

```
  @IsString()
  @IsNotEmpty()

  @IsNumber()

  @IsOptional()
```


# DTO and mapped types

NestJS provides helpers to create update DTOs from create DTOs.

Most famous one is:

`PartialType()`

Install if needed:

```
npm install @nestjs/mapped-types

```

`PartialType()`

Suppose CreateUserDto has many fields.
For update, usually same fields are needed, but all should be optional.

`create-user.dto.ts`

```
import { IsEmail, IsNotEmpty, IsString, MinLength } from 'class-validator';

export class CreateUserDto {
  @IsString()
  @IsNotEmpty()
  name: string;

  @IsEmail()
  email: string;

  @IsString()
  @MinLength(6)
  password: string;
}

```

`update-user.dto.ts`

```
import { PartialType } from '@nestjs/mapped-types';
import { CreateUserDto } from './create-user.dto';

export class UpdateUserDto extends PartialType(CreateUserDto) {}

```
It makes all fields optional.

NestJS also provides:
- `PickType()`
- `OmitType()`
- `IntersectionType()`


`PickType()` example:
Take only selected fields from another DTO.

```
import { PickType } from '@nestjs/mapped-types';
import { CreateUserDto } from './create-user.dto';

export class LoginDto extends PickType(CreateUserDto, ['email', 'password'] as const) {}

```

`OmitType()` example
Take all fields except some fields.

```
import { OmitType } from '@nestjs/mapped-types';
import { CreateUserDto } from './create-user.dto';

export class PublicUserDto extends OmitType(CreateUserDto, ['password'] as const) {}

```
This excludes password.

`IntersectionType()` example

Combine two DTOs.

```
import { IntersectionType } from '@nestjs/mapped-types';

class PersonalInfoDto {
  name: string;
  email: string;
}

class AddressDto {
  city: string;
  country: string;
}

export class FullUserDto extends IntersectionType(PersonalInfoDto, AddressDto) {}

```

# NestJS Pipes

1. What is a Pipe in NestJS?
  A Pipe in NestJS is a class that is used to:
  - transform data before it reaches your controller method
  - validate data before your controller logic runs


A pipe sits between the incoming request and your controller method.

We use pipes to make sure the data coming from the client is correct and usable.


2. Where Pipes are used

Pipes are commonly used with:
- `@Param()`
- `@Query()`
- `@Body()`
- sometimes custom decorators too


example:

```

@Get(':id')
findOne(@Param('id', ParseIntPipe) id: number) {
  return this.usersService.findOne(id);
}

```

3. Types of Pipes in NestJS

 A. Transformation Pipes
  - These change the incoming data type.

 B. Validation Pipes
  - These check whether the value is valid.


4. Built-in Pipes in NestJS
   NestJS provides some ready-made pipes.

  - `ValidationPipe`
  - `ParseIntPipe`
  - `ParseBoolPipe`
  - `ParseArrayPipe`
  - `ParseUUIDPipe`
  - `DefaultValuePipe`

a. ParseIntPipe
- It converts a string into an integer.
- If conversion fails, it throws an error.

example:

```
import { Controller, Get, Param, ParseIntPipe } from '@nestjs/common';

@Controller('users')
export class UsersController {

  @Get(':id')
  getUser(@Param('id', ParseIntPipe) id: number){

    return {
      message: 'User fetched successfully',
      id,
      type: typeof id,
    };

  }

}


```

`GET /users/5`

```
{
  "message": "User fetched successfully",
  "id": 5,
  "type": "number"
}

```

`GET /users/abc`

```
{
  "statusCode": 400,
  "message": "Validation failed (numeric string is expected)",
  "error": "Bad Request"
}

```

b. ParseBoolPipe
It converts a string value to boolean.

example:

```
import { Controller, Get, Query, ParseBoolPipe } from '@nestjs/common';

@Controller('products')
export class ProductsController {

  @Get()
  getProducts(@Query('active', ParseBoolPipe) active: boolean) {

    return {
      active,
      type: typeof active,
    };

  }
}

```

`GET /products?active=true`

```
{
  "active": true,
  "type": "boolean"
}

```

c. ParseUUIDPipe

It checks whether the given value is a valid UUID.

```
import { Controller, Get, Param, ParseUUIDPipe } from '@nestjs/common';

@Controller('orders')
export class OrdersController {
  @Get(':id')
  getOrder(@Param('id', ParseUUIDPipe) id: string) {
    return { id };
  }
}

```

If invalid UUID is passed, NestJS throws error.


d. DefaultValuePipe
It gives a default value if the client does not send one.

```

import { Controller, Get, Query, DefaultValuePipe, ParseIntPipe } from '@nestjs/common';

@Controller('items')
export class ItemsController {
  @Get()
  getItems(
    @Query('page', new DefaultValuePipe(1), ParseIntPipe) page: number,
  ) {
    return { page };
  }
}

```

`GET /items`

```
{
  "page": 1
}

```
d. ValidationPipe

This is one of the most important pipes in NestJS.
It validates request body, query, or params using DTO classes.

Usually used with:
- `class-validator`
- `class-transformer`



DTO and ValidationPipe connection

DTO = `Data Transfer Object`
It defines the expected structure of incoming data.

```
import { IsEmail, IsNotEmpty, IsString, MinLength } from 'class-validator';

export class CreateUserDto {
  @IsString()
  @IsNotEmpty()
  name: string;

  @IsEmail()
  email: string;

  @MinLength(6)
  password: string;
}

```

Now use it in controller:

```
import { Body, Controller, Post, UsePipes, ValidationPipe } from '@nestjs/common';
import { CreateUserDto } from './dto/create-user.dto';

@Controller('users')
export class UsersController {

@Post()
@UsePipes(new ValidationPipe())
createUser(@Body() createUserDto: CreateUserDto) {
    return {
      message: 'User created successfully',
      data: createUserDto,
    };
}



}





```

valida data body

```
{
  "name": "Sangam",
  "email": "sangam@gmail.com",
  "password": "123456"
}

```

```

{
  "name": "",
  "email": "wrong-email",
  "password": "123"
}

```

Global ValidationPipe
Instead of using @UsePipes() in every controller, we usually apply ValidationPipe globally.

```
import { ValidationPipe } from '@nestjs/common';
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  app.useGlobalPipes(
    new ValidationPipe({
      whitelist: true,
      forbidNonWhitelisted: true,
      transform: true,
    }),
  );

  await app.listen(3000);
}
bootstrap();

```


`whitelist: true`
Removes extra properties that are not defined in DTO.

`forbidNonWhitelisted: true`
Instead of silently removing extra fields, it throws an error.
This is more strict and secure.

`transform: true`
Automatically transforms plain input into DTO object and converts types if possible.
Very useful when working with DTO classes.



Create custom pipe
You can also create your own pipe.

A custom pipe is a pipe that you create yourself when NestJS built-in pipes are not enough.

```

```


`src/status-validation.pipe.ts`
`src/status-validation.pipe.spec.ts`

defult code 

```
import { ArgumentMetadata, Injectable, PipeTransform } from '@nestjs/common';

@Injectable()
export class StatusValidationPipe implements PipeTransform {
  transform(value: any, metadata: ArgumentMetadata) {
    return value;
  }
}

```

```

import {
  ArgumentMetadata,
  Injectable,
  PipeTransform,
  BadRequestException,
} from '@nestjs/common';

@Injectable()
export class StatusValidationPipe implements PipeTransform {
  transform(value: any, metadata: ArgumentMetadata) {
    const allowed = ['active', 'inactive'];

    if (!allowed.includes(value)) {
      throw new BadRequestException('Invalid status');
    }

    return value;
  }
}

```

Where pipes can be applied

Pipes can be applied at different levels:

Parameter level

```
@Param('id', ParseIntPipe) id: number

```

Method level

```
@UsePipes(new ValidationPipe())
@Post()
create(@Body() dto: CreateUserDto) {}

```

Controller level

```
@UsePipes(new ValidationPipe())
@Controller('users')
export class UsersController {}

```

Global level

```
app.useGlobalPipes(new ValidationPipe());

```


# NestJS Dependency Injection (DI)

Dependency Injection is a design pattern where one class gets the objects it needs from outside, instead of creating them itself.

A class should not manually create its required dependencies.
NestJS gives them automatically.


Basic example of DI

example:

`users.service.ts`

```
import { Injectable } from '@nestjs/common';

@Injectable()
export class UsersService {
  getAllUsers() {
    return ['Sangam', 'Rahul', 'Amit'];
  }
}

```

```
import { Controller, Get } from '@nestjs/common';
import { UsersService } from './users.service';

@Controller('users')
export class UsersController {
  constructor(private readonly usersService: UsersService) {}

  @Get()
  findAll() {
    return this.usersService.getAllUsers();
  }
}

```

`@Injectable()` tells NestJS that this class can be managed by the NestJS DI container.

```
@Injectable()
export class UsersService {}

```

NestJS looks at the constructor. for injection

```
constructor(private readonly usersService: UsersService) {}

```

A route parameter is a value that comes from the URL path.

A value passed inside the URL that changes for each request.

`GET /users/10`


```

/users/1
/users/25
/products/100
/orders/500



```

In NestJS, parts of the route can be **fixed** or **dynamic**:

- **Fixed part** → e.g., `users`
- **Dynamic part** → e.g., `10` (this is called a **route parameter**)


example

```
import { Controller, Get, Param } from '@nestjs/common';

@Controller('users')
export class UsersController {
  @Get(':id')
  findOne(@Param('id') id: string) {
    return `User ID is ${id}`;
  }
}

```

What this means

`@Controller('users')`

```
/users

```

`@Get(':id')`

This means the route is dynamic.

So it can match:
```
GET /users/1
GET /users/2
GET /users/99

```

Step-by-step flow

Suppose request is:

```
GET /users/10

```

Step 1

NestJS sees controller base route:

```
@Controller('users')

```

So it matches `/users`

Step 2

NestJS sees method route:

```
@Get(':id')

```
So after /users, it expects one dynamic value


Step 3

URL is /users/10

So NestJS understands:

`id = "10"`

Step 4

`@Param('id')` reads that value

Step 5

Method receives:

```
id = "10"

```

Step 6

Response becomes:
User ID is 10

```
import { Controller, Get, Param } from '@nestjs/common';

@Controller('users')
export class UsersController {
  @Get(':id')
  getUserById(@Param('id') id: string) {
    return {
      message: 'User fetched successfully',
      userId: id,
    };
  }
}

```
request

`GET /users/5`


Response:

```
{
  "message": "User fetched successfully",
  "userId": "5"
}
```


Route parameters come from URL, so NestJS reads them as string by default.
